What's new in z/OS V3R1 RMF

Table 1. Whats New in RMF V3R1
What's New summary
The RMF XML Toolkit is not supported in z/OS V3R1 RMF.
Shared customization of Monitor III options and reports
  • Monitor III Options, Option Sets, and Reports can be customized and stored in a common RMF table library to be shared among RMF users.
Increased LPAR usage precision in CPC report
  • The LPAR usage precision is increased to 2 decimal places in the CPC report.
Additional XCF reports to Monitor III SYSPLEX reports menu:
  • XCFOVW - XCF Overview
  • XCFSYS - XCF Systems data
  • XCFPATH - XCF Path statistics
  • XCFGROUP - XCF Group data
Improved Monitor III STORC report

The MON III STORC report is modified to separate RUCSA from CSA and ERUCSA from ECSA. The new fields are not visible by default. Use RMF UTIL to customize the existing reports or to create new reports to display these data fields.

  • New columns are created for RUCSA and ERUCSA
  • The values reported in the CSA column no longer includes RUCSA
  • The values reported in the ECSA column no longer includes ERUCSA
Inclusion of Enclave Transaction metrics:
Monitor III
New data fields are made available to SYSSUM and SYSINFO reports. The fields are not visible by default. Use RMF UTIL to customize the existing reports or to create new reports to display these data fields.
  • SYSSUM
    • SUMECTR - Enclave transaction rate per sec
    • SUMECTRE - Average enclave transaction execution time in milli-seconds
  • SYSINFO
    • SYSECTVC - Enclave transaction rate per sec
    • SYSECEVC - Average enclave transaction execution time in milli-seconds
Post-processor
New columns are added to the Response Time and Resource Consumption Data section of the WLMGL report.
  • ENCLAVE TRANS
    • AVG - The average number of executing enclave bundled transactions during the interval
    • END/S - Enclave transactions per second
    • ENC DEL - Number of times transaction data provided with enclave delete
    • TR/ENC - Average number of enclave transactions per enclave
  • ENC-TR-TM
    • EXECUTION - Average execution time of enclave transactions
    • EX STD DEV - Standard deviation of enclave transaction execution time
Spreadsheet Reporter
The following spreadsheet range names are added.
  • ENCLAVE TRANS
    • AVG "ENCTRAVG"
    • END/S "ENCTRRTE"
    • ENC DEL "ENCTRDEL"
    • TR/ENC "ENCTRENC"
  • ENC-TR-TM
    • EXECUTION "ENCTRTEX"
    • EX STD DEV "ENCTRESD"
System Recovery Boost for Middleware
RMF is enhanced to report on Middleware Address spaces that are boosted to reduce recovery time.
Monitor III
  • The Monitor III PROCU report is modified to display Recovery Process Boost in a column called "Rcvy Boost". Table ERBPRUT3 contains a new field called PRUPRPB.
  • An optional field is also added to the USAGE report to display Recovery Process Boost status. Table ERBJUST3 contains a new field JUSPRPB. The new field is not visible by default. Use RMF UTIL to customize existing reports or to create new reports to display this data field.
Monitor II and Post-processor
The ARD and ASD commands have been modified to accept an optional parameter to filter the report by boost status.
  • A - All address spaces are included (this is the default)
  • B - Only Boosted - Only address spaces which have been boosted are included
  • In ISPF, boosted address spaces are highlighted with reversed turquoise
SMF Record Up/Down Conversion for Realtime Reporting
  • RMF Realtime Reporting has been enhanced to support Up/Down Conversion of SMF records that were written by a different RMF version than that of the current system.
Support for z16 Hardware and Crypto Express8s for z16
  • Support has been added for z16 Hardware and for Crypto Express8s for z16.
Improved performance of XCF report in Spreadsheet Reporter when processing large amounts of data
  • Spreadsheet Reporter performance has been improved when processing very large amounts of data to produce the XCF report.

Configuration setting to enable/disable alphabetical sorting of tabs in PM Client

  • A configuration setting has been added to the PM Client to control whether tabs are sorted alphabetically. When this setting is disabled, the tabs order is not specified.
